gort is a go reload thingy. don't use this

gort takes in a toml(gott) file with reload settings. One toml table correlates to one template+file.

Example gort usage with bspwm follows.

The toml:

#+begin_src conf-toml
# /path/to/theme.toml
[bspwm]
border-width = 4
#+end_src

The template:

#+begin_src sh
#!/bin/sh
bspc config @{border-width}
#+end_src

The gort config:

#+begin_src conf-toml
# global table:
sources = [
    "/path/to/theme.toml",
    "/path/to/another/gott/file.toml"
]

# A loader for a file
[bspwm]
destination = "$HOME/.config/bspwm/bspwmrc"
template = "$HOME/.dotfiles/theming/templates/bspwm"

# contexts are used to promote tables from sources to top level
contexts = [ "bspwm" ]

# reload command
reload = """
chmod +x @{destination}
@{destination}
"""
#+end_src

The gort command:

#+begin_src sh
gort -c '/path/to/gort.toml' -t bspwm
#+end_src

This will render the template to the destination, and execute the value of ~bspwm.reload~ using ~GORT_SHELL~ (default ~bash~)
